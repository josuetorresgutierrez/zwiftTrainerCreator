# Prompt para Stitch

Diseñá la interfaz final de una app web llamada **Zwift Workout Composer**.

## Contexto
- La app transforma entrenamientos escritos en texto o DSL en workouts estructurados compatibles con Zwift.
- El foco no es una UI bonita genérica, sino una herramienta técnica, rápida y profesional para ciclistas y entrenadores.
- Los mockups de referencia están en la carpeta `stich_zwift_trainer_creator_mockups`.
- Quiero que uses esos mockups como base visual, pero que mejores la UX, la jerarquía y la claridad del flujo.

## Objetivo del diseño
Crear una experiencia final que permita:
1. Crear entrenamientos.
2. Validarlos en tiempo real.
3. Ver una previsualización clara del workout.
4. Exportar a Zwift.

## Dirección visual
- Estética dark premium, industrial, tipo telemetry/cockpit.
- Mantener el naranja como color de acento principal.
- Superficies oscuras, contraste alto y tipografía contundente.
- Bordes rectos, grilla precisa, composición limpia.
- Evitar decorar por decorar: priorizar legibilidad, orden y velocidad de uso.

## Lo que quiero mejorar respecto a los mockups
- El sidebar debe pesar menos visualmente o poder colapsarse.
- El editor debe aprovechar mejor el espacio y mostrar ayudas contextuales.
- La validación debe ser inmediata y visible.
- La exportación debe sentirse como el final natural del flujo.
- La app debe verse consistente entre dashboard, editor y exportación.
- Debe haber estados vacíos, de carga y de error.
- La UI debe funcionar bien en desktop y también adaptarse a pantallas medianas.

## Pantallas a diseñar
### 1. Dashboard
- Resumen de métricas clave.
- Acceso rápido a “Nuevo entrenamiento”.
- Plantillas recientes.
- Sesiones recientes.
- CTA principal claro.

### 2. Editor de entrenamiento
- Layout split-pane.
- Izquierda: editor DSL/texto con numeración de líneas, ayuda contextual y errores inline.
- Derecha: gráfico de potencia y preview del workout en tiempo real.
- Mostrar FTP, sintaxis, repeticiones y bloques interpretados.

### 3. Análisis y exportación
- Métricas del workout.
- Gráfico principal legible.
- Estado de validación.
- Botón de exportación dominante pero bien jerarquizado.

## Requisitos UX
- Flujo claro: crear -> validar -> previsualizar -> exportar.
- Feedback visual inmediato.
- Microcopy breve y útil.
- Mantener foco en la tarea, no en adornos.
- Las acciones principales deben estar siempre accesibles.

## Entregable esperado
- Un diseño final pulido y coherente.
- Un sistema visual consistente.
- Mejor jerarquía visual.
- Mejor uso del espacio.
- Una interfaz lista para implementar.

## Restricciones
- No inventes funcionalidades fuera del objetivo del producto.
- No hagas una UI decorativa que sacrifique claridad.
- No pierdas el carácter técnico del producto.
- El resultado debe sentirse como un traductor inteligente de entrenamientos hacia Zwift.

## Resultado deseado
Generá una propuesta final de diseño para toda la app, manteniendo la estética telemetry pero con una UX mucho más sólida, usable y lista para pasar a desarrollo.
